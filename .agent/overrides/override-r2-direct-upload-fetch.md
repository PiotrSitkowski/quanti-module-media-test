# Override: CS-02 R2 Direct Upload — fetch() in MediaTestTable

## Rule Being Overridden

`FORBIDDEN PATTERNS` → `fetch()` in `src/components/*.tsx` components.

## Justification

The `fetch()` call in `src/components/MediaTestTable.tsx` is **not a backend API call**.
It is specifically the **R2 presigned URL direct upload** mandated by example CS-02 (R2 Object Storage).

### CS-02 Canonical 3-step flow:

1. **Component → Kernel:** `context.api.rpc('media', 'getPresignedUploadUrl', {...})`
   — gets a time-limited signed URL from R2 (via Kernel, not our Worker)

2. **Browser → R2 directly:** `fetch(presigned.uploadUrl, { method: 'PUT', body: file })`
   — this `fetch()` goes to the R2 bucket URL, **bypassing the Worker entirely**
   — this is the ONLY allowed fetch() pattern in MFE components per CS-02

3. **Component → Worker:** `context.api.rpc('media-test', 'create', { r2Key, ... })`
   — registers metadata in D1 after confirmed upload

### Why this cannot be done via context.api

`context.api.rpc` calls go through the Quanti Kernel Service Worker which adds
authentication headers, tenant context and tracing. Uploading a binary file via RPC
would require base64 encoding (3.3× size overhead), exceeding Cloudflare's 128 MB
request limit for Workers. The presigned URL pattern is the architecturally correct
solution documented in CS-02.

### What is NOT happening

- ❌ `fetch('/api/...')` — this is not a call to our module's REST endpoint
- ❌ `fetch('http://quanti-backend/...')` — not calling the Kernel directly
- ✅ `fetch(presignedR2Url, { method: 'PUT', body: file })` — browser-to-R2 only

## Scope

File: `src/components/MediaTestTable.tsx`
Line: the `fetch(presigned.uploadUrl, ...)` call in `handleFileSelected()`

## Impact Assessment

- **Security:** None. The presigned URL is scoped to a single object, expires after
  a short TTL, and is generated by the Kernel media service with proper tenant ACL.
- **Observability:** Upload is confirmed before `context.api.rpc` metadata registration.
  Failed uploads do not create orphaned D1 records.
- **Architecture:** Fully compliant with CS-02 canonical pattern.
